# Project Constitution: Smart Emergency Resource Locator

## 1. Principles
- **Data Accuracy**: Reliability of resource information is paramount.
- **Accessibility**: The application must be accessible on all devices.
- **User Privacy**: No PII collected.

## 2. Tech Stack
- **Frontend**: Streamlit
- **Backend**: Python
- **Data**: CSV

## 3. Development Guidelines
- **Code Quality**: Follow PEP 8 (enforced via Ruff).
- **Type Safety**: Use strict type hinting (enforced via MyPy).
- **Automation**: Pre-commit hooks must be used for all code changes.
- **Spec-Driven Development**: All features require a comprehensive `spec.md`, `plan.md`, and `tasks.md` defined in `specs/`.
